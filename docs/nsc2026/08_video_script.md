## สคริปต์ภาษาไทย (พูดจริง)

### ฉากที่ 1 (0:00 – 0:45) — Hook

> *[ภาพ: SMS แจ้งเตือนจากกรมสรรพากร พร้อมลิงก์]*
>
> "คุณป้าจินตนาเปิดมือถือดู เจอ SMS แจ้งจากกรมสรรพากรว่ามีเงินคืนภาษี
> ให้คลิกลิงก์เพื่อรับเงิน 8,400 บาทใน 24 ชั่วโมง คุณป้าเห็นลิงก์ `rd-go-th.refund.online`
> ลงท้ายด้วย `.online` แต่มีคำว่า `rd` กับ `go` กับ `th` คุณป้าเลยคลิกแล้ว
> กรอกเลขบัตรประชาชน เลขบัญชี รหัส OTP...
>
> *[เปลี่ยนเป็นภาพข่าวจริงเรื่อง scam]*
>
> ปีที่แล้วคนไทยเสียเงินให้กับ phishing แบบนี้ปีละหลายพันล้านบาท ETDA
> รายงานกว่า 30,000 เคส ผู้เสียหายส่วนใหญ่คือผู้สูงอายุและคนที่ไม่ชินกับ
> เทคโนโลยี — และระบบป้องกันระดับโลกที่มีอยู่ พลาด URL ปลอมแบรนด์ไทย
> เพราะไม่มี whitelist และข้อมูลเฉพาะของไทย"

### ฉากที่ 2 (0:45 – 1:00) — แนะนำตัว

> "สวัสดีครับ ผม/ดิฉัน ___________ นักเรียนชั้น ___________ จาก
> ___________ ภายใต้การดูแลของอาจารย์ ___________ ขอนำเสนอโครงการ
> 'ระบบตรวจจับเว็บไซต์ฟิชชิงสำหรับหน่วยงานราชการและสถาบันการศึกษาไทย'
> ภายใต้หมวด 23 ระดับนักเรียน"

### ฉากที่ 3 (1:00 – 1:45) — Overview

> *[ภาพ: architecture diagram ที่เคลื่อนไหวทีละชิ้น]*
>
> "เราออกแบบระบบนี้เป็นแพลตฟอร์มที่ทำงานร่วมกัน 5 ส่วน หนึ่ง ML pipeline
> ที่ฝึกโมเดล RandomForest + XGBoost บน 37 features สอง backend ภาษา
> Python ที่มี API สำหรับตรวจ URL และส่งข้อมูลให้แอป สาม ส่วนขยายเบราว์เซอร์
> Manifest V3 ที่ใช้ได้ในทุกเบราว์เซอร์หลัก สี่ dashboard React 11 หน้า
> สำหรับ operator และห้าที่สำคัญที่สุด — citizen report portal ที่ใช้ได้
> โดยไม่ต้อง login เพื่อให้ accessibility สูงสุด"

### ฉากที่ 4 (1:45 – 2:30) — Demo Extension

> *[ภาพ: เปิด Chrome → พิมพ์ URL ปลอม → ทันทีหน้าแดงโผล่]*
>
> "ผมจะ demo การใช้งานเลยนะครับ ผมพิมพ์ URL ที่เลียนแบบสำนักงานคณะกรรมการ
> การศึกษาขั้นพื้นฐาน — `obec.com/verify-account` — ก่อนหน้าเว็บจะโหลด
> ส่วนขยายเราตรวจ URL กับ backend และเจอว่าเป็นฟิชชิง หน้าเตือนเต็มจอ
> จะโผล่ทันที พร้อมเหตุผลเป็นภาษาไทยและปุ่มกลับหน้าก่อนหน้า ผู้ใช้ที่รู้
> ตัวเองว่ากำลังทำอะไรอยู่สามารถ bypass ได้ — แต่ bypass นี้จะหมดอายุ
> เมื่อเปิด browser ใหม่ครั้งหน้า"

### ฉากที่ 5 (2:30 – 3:00) — Demo Citizen Portal

> *[ภาพ: เปิดหน้า /report → พิมพ์ URL → เห็น verdict → กดส่ง]*
>
> "ถ้าคุณป้าจินตนาไม่ได้ติดตั้งส่วนขยาย เธอสามารถพิมพ์ URL ที่สงสัยลงในหน้า
> 'แจ้งเว็บฟิชชิง' ของระบบเรา — ไม่ต้อง login — ระบบจะตรวจให้ทันที พร้อม
> แสดง rule ที่ทำงาน ถ้าผลผิด เธอสามารถระบุว่าคิดว่าเป็นอะไรแล้วส่งกลับให้
> เรา feedback นี้จะใช้ปรับโมเดลในรอบ retrain ครั้งถัดไป"

### ฉากที่ 6 (3:00 – 4:00) — เทคนิค

> *[ภาพ: code snippet + feature list + rules engine output]*
>
> "เทคนิคที่ใช้มีสองชั้นทำงานร่วมกัน ชั้นแรกคือ ML — เราออกแบบ feature
> เอง 37 ตัว ที่สำคัญที่สุดคือกลุ่ม IDN/Homoglyph ที่ decode Punycode
> และ fold Unicode confusables จับการปลอมตัวอักษรซีริลลิกได้ และ
> feature ใหม่ใน v1.4.0 ล่าสุด — num_login_keywords, query_param_count,
> path_entropy, host_token_count — จับ phishing kit แบบ
> 'random host + brand ใน path' ที่เป็น pattern หลักของปี 2024-2025
>
> ชั้นที่สองคือ Rules Engine — กฎที่เขียนเป็น Python ฟังก์ชัน แต่ละกฎมี
> rule_id, score delta และ pin label ทุก verdict ที่ออกจากระบบจะมี
> rules.hits ที่บอกว่ากฎไหนทำงาน เพิ่มความโปร่งใส, ตรวจสอบได้, และ
> operator สามารถ override ML ได้โดยไม่ต้อง retrain"

### ฉากที่ 7 (4:00 – 4:45) — Demo Watchlist + LINE

> *[ภาพ: เปิด dashboard watchlist page → กรอก brand + LINE token →
> ใช้ extension เปิด URL ปลอม → LINE มี notification ภายในไม่กี่วินาที]*
>
> "สำหรับหน่วยงานราชการ เราออกแบบ Brand Watchlist — ลงทะเบียน 'krungthai'
> พร้อม LINE Notify token หนึ่งครั้ง เมื่อมีคนตรวจ URL ที่ปลอม krungthai
> จาก browser extension ทั่วประเทศ ระบบจะ POST alert ไปยัง LINE ของ
> หน่วยงานภายในไม่กี่วินาที พร้อม retry และ delivery log"

### ฉากที่ 8 (4:45 – 5:15) — ความแม่นยำ

> *[ภาพ: graph confusion matrix + roc curve + 100% recall headline]*
>
> "เราวัดความแม่นยำบน Thai-targeting holdout 66 URL ที่โมเดลไม่เคยเห็น
> ระหว่าง training — ได้ recall 100% (66/66) และ generic phishing holdout
> 98.9% (89/90) ระบบมี CI gate ขั้นต่ำ 0.85 ทุก commit ต้องผ่าน
> automated tests 206 cases ทั้งหมดผ่าน"

### ฉากที่ 9 (5:15 – 6:15) — Sustainable Innovation

> *[ภาพ: 4 pillars + numbers ของ /api/v1/impact]*
>
> "ระบบนี้ออกแบบให้ตอบโจทย์ Sustainable Innovation ของ NSC 2026 ใน 4 มิติ
> พร้อมกัน หนึ่ง social — citizen portal ไม่ต้อง login, extension ฟรี,
> เนื้อหา /learn เป็นภาษาไทย สอง economic — endpoint /impact คำนวณ
> ยอดเสียหายที่ป้องกันได้แบบเรียลไทม์โดยอ้าง median ของ ETDA สาม
> environmental — สถาปัตยกรรม low-resource ไม่ใช้ GPU, deploy บน
> VPS ราคาประหยัดได้ และสี่ open — ทั้งโค้ดและ dataset เปิดภายใต้ Apache 2.0
> public threat feed เป็น STIX 2.1 ใครก็ดึงได้"

### ฉากที่ 10 (6:15 – 6:45) — Roadmap

> "ในระยะ 3 เดือนต่อไป เราจะขยาย seed corpus, สร้าง LINE bot ให้คน
> ทั่วไปพิมพ์ URL ตรวจในแชต และทดสอบกับ pilot user จริง — โรงเรียน 2 แห่ง
> และหน่วยงานราชการ 1 แห่ง — เพื่อยืนยันผลกระทบทางสังคม"

### ฉากที่ 11 (6:45 – 7:00) — Closing

> "ทุกอย่างของระบบนี้เปิดเป็น open source อยู่ที่ GitHub link ด้านล่าง
